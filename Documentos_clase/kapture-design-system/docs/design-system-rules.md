# Design System Rules — KAPTURE

## Regla 1: Jerarquía Visual
- Lo más importante debe destacar primero (tamaño, peso, contraste, espacio).
- Un objetivo por sección.

## Regla 2: Fricción Intencional
- Acciones críticas deben pedir confirmación o ser más “lentas” para evitar errores.
- Acciones comunes deben ser rápidas y claras.

## Regla 3: Consistencia de Estados
- Todos los patrones deben definir: default, hover, pressed, focus, disabled, error (si aplica).
- El mismo estado debe verse igual en todo el sistema.
